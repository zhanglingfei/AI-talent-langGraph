LANGGRAPH WORKFLOW VISUALIZATION

IT Talent Matching System Based on LangGraph

Graph State Variables

current_email: EmailInfo
email_type: EmailType Enum
classification_confidence: float (0-1)
candidate_info: CandidateInfo
project_info: ProjectInfo
errors: List[str]
processing_log: List[str]
retry_count: int

Processing Flow Description

  • ENTRY POINT: Receives email data (EmailInfo object)
  • CLASSIFICATION NODE: Uses GPT-4 to analyze email type (Candidate/Project/Other)
  • CONDITIONAL ROUTING: Routes based on classification result and confidence score
  • INFORMATION EXTRACTION: Parses candidate or project info using Pydantic models
  • DATA PERSISTENCE: Saves to Google Sheets database

Graph State Variables

match_type: "project_to_resume" | "resume_to_project"
match_query_id: string
prefiltered_items: List[Dict]
match_results: List[MatchResult]
errors: List[str]
processing_log: List[str]

Matching Flow Description

  • CONDITIONAL ENTRY: Routes based on match_type parameter
  • PREFILTERING: Initial keyword-based filtering
  • AI MATCHING: GPT-4 deep analysis and scoring
  • RESULT STORAGE: Saves match results and scores

System Architecture Description

  • DATA SOURCE: Gmail API for email and attachment retrieval
  • PROCESSING PIPELINE: Email classification → Information extraction → Data storage
  • MATCHING ENGINE: Prefiltering → AI deep matching → Result storage
  • DATA STORAGE: Google Sheets as persistence layer
  • LLM INTEGRATION: GPT-4 for classification, extraction, and matching
